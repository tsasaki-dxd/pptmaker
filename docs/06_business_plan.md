## 2. 事業仮説

### 2.1 バリュープロポジション

- 提案書作成時間の **短縮**（現状 6 時間 → 2 時間目標）
- コーポレートテンプレート準拠の **品質均質化**
- 属人性の排除・ナレッジの蓄積（テンプレ／図表パターン）
- 日本語・日本のビジネス文脈への **最適化**

### 2.2 想定スケール（初期）

| Phase | ユーザー規模 | 月間生成数 |
|---|---:|---:|
| Phase 1（社内） | 50 名（営業 30 / コンサル 20） | 200 件 |
| Phase 2（顧客案件） | 5 社 × 50 名 | 800 件 |
| Phase 3（SaaS） | 50 社 × 30 名 | 6,000 件 |

| フォント / ソフト | Noto CJK = 無償、LibreOffice = 無償 | 0 |
| **合計** | | **約 19 万円** |

予備・バースト対応込みで **月 25〜30 万円**を予算化。

### 4.2 Claude API コスト試算

想定（`01_prompt_engineering.md` §8 より、1 提案書あたり）:
- input 17,500 tokens / output 6,500 tokens
- Prompt Caching 70% ヒット想定 → 有効 input ≒ 5,250 + 12,250×0.1 = 6,475
- Sonnet 4.6 想定単価：input $3 / output $15 / MTok（仮）
- 1 提案書：($3 × 6,475 + $15 × 6,500) / 1M ≒ $0.117
- 月 200 提案書：約 $23 → 約 3,500 円
- 修正リトライ・Eval・内部ツール込みで実績バッファ 10 倍の **月 35,000 〜 55,000 円** を見積もり

## 7. ROI 分析（Phase 1）

### 7.1 現状

- 1 提案書平均作成時間：6 時間
- 営業担当の時間単価：7,500 円 / 時（600 万 / 年 × 係数 1.5 / 1,200 時）
- 月 200 件 × 6 時間 × 7,500 円 = **月 900 万円 の労務投下**

### 7.2 導入後

- 1 提案書平均 2 時間（目標）
- 月 200 件 × 2 時間 × 7,500 円 = 300 万円
- **時間削減価値：月 600 万円**

### 7.3 回収

- 月次ランニング 30 万円 vs 削減 600 万円
- 開発費 Phase 0〜1 合計 1,182 万円
- 回収期間 ≒ 2 ヶ月（稼働開始後）

当然前提条件の精度は要検証。先行 10 件でタイムトラッキングを実施して更新する。

## 8. 競合分析

| サービス | 強み | 弱み |
|---|---|---|
| Microsoft 365 Copilot | PowerPoint 統合・既存資産活用 | 日本語品質・コーポレートテンプレ準拠の細かさが弱い、カスタム図表は限定 |
| Tome | 演出的プレゼン | 堅めのビジネス提案書には不向き |
| Gamma | 速い生成、編集しやすい | ブランドガバナンス弱、.pptx 互換に課題 |
| Beautiful.ai | 洗練テンプレ | 英語中心、テンプレ持ち込み制約 |
| SlideSpeak | ファイル中心の変換 | UI がやや簡素 |

**SlideForge の差別化**

1. コーポレートテンプレート**持ち込み準拠**（正規化＋プロファイル化）
2. 日本語特化・日本の提案書文化（セクション扉、会社概要、免責事項、金額表）
3. 図表プラグイン IF による**業務特化拡張**
4. 自社の社内業務での実績を顧客に提示できる

## 9. リスクとサニティチェック（ビジネス観点）

| # | リスク | 影響 | Mitigation |
|---|---|---|---|
| B1 | Copilot が無償バンドル化 | SaaS 化の価値毀損 | コーポレート持込・拡張性・日本語で差別化、大企業向けオンプレ路線を担保 |
| B2 | Claude API 価格上昇 / Rate Limit | 原価悪化 | モデル切替（Haiku 化）、Prompt Caching 強化、複数ベンダー対応余地を設計 |
| B3 | 社内利用が定着しない | Phase 1 失敗 | 先行ユーザー 5 名にデザインパートナー制、月次利用レビュー |
| B4 | 顧客テンプレが想定外パターン | 対応不可 | テンプレ登録段階で非対応を早期検知、段階的対応 |
| B5 | 情報漏洩事故 | 信用毀損 | セキュリティ設計書 §14 体制、Phase 3 前に外部監査 |
| B6 | 人員確保難 | 開発遅延 | Phase 2 以降で外部パートナー・オフショア活用案 |

## 10. ロードマップ（肉付け）

### 10.1 Phase 0（PoC 整理、2 週間）

- W1: 既存スクリプトのモジュール分割、テンプレプロファイルのスキーマ確定、Eval 初期セット 30 件
- W2: CI セットアップ（lint / unit / VRT の骨格）、Phase 1 要件確定、キックオフ準備

### 10.2 Phase 1 MVP バックログ（MoSCoW）

| 優先度 | 項目 |
|---|---|
| Must | Cognito 認証、テンプレ登録＋プロファイル、骨格生成（自由記述）、プレビュー、.pptx エクスポート、ロギング基盤 |
| Must | PowerPoint 互換 CI（Lint）、監査ログ、テナント分離 |
| Should | 構造化フォーム入力、修正指示（テキスト修正まで）、履歴閲覧 |
| Could | 既存文書取込、色調調整、差分プレビュー、チャット |
| Won't | 共同編集、モバイル、音声入力 |

### 10.3 Go/No-Go ゲート

| 判定 | ゲート基準 |
|---|---|
| Phase 1 → 2 | WAU ≥ 30 / 50、1 提案書所要時間 ≤ 3 時間、NPS ≥ 20、重大セキュリティ事故ゼロ |
| Phase 2 → 3 | 顧客案件 2 件以上導入、SLO 達成、ペネトレーションテスト合格、損益分岐の見通し |
| Phase 3 継続 | MRR 増加、解約率 ≤ 5% / 月、SOC 2 Readiness 合格 |

### 10.4 マイルストーン（目安）

| 時期 | マイルストーン |
